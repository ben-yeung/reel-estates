# Consultation Booking is deep-linkable via `?book=`, and existing CTAs point at it

Consultation Booking (ADR 0005) auto-fills an Agent from a selected Property, which raises an opportunity: the site already carries property context in the URL, and already has "Book Consultation" affordances that lead nowhere. The Property Detail Modal's agent sidebar has a decorative "Book Consultation" button, and Agent surfaces exist that could reasonably offer the same. Wiring these to pre-fill the booking turns Consultation Booking from an island into the funnel floor for the whole site: a visitor deep in a property or agent context can jump straight into a booking with that context already applied.

We do this with a dedicated `?book=[slug]` query param, **not** the modal's existing `?property=[slug]`. Reusing `?property=` would re-open the Property Detail Modal (that param is what drives the modal open - see ADR 0003), which is the opposite of what a "book a consultation" CTA should do. So the two params have disjoint jobs: `?property=` opens the modal, `?book=` pre-selects a Property in the Consultation Booking. On load, the section reads `?book=`, and if present pre-selects that Property (and thus its Agent) and lands the visitor on the section. The "Book a consultation" CTAs - in the modal's agent sidebar and on Agent cards - set `?book=[slug]` (closing the modal where applicable) and scroll to `#contact`. The general-enquiry path is simply the absence of `?book=`.

We considered a smaller scope - read the param but hold off on scattering CTAs, or keep the section fully self-contained for v1 and defer all cross-linking to a later ADR. We chose the full wiring because the section's whole value proposition is context-aware booking, and shipping it with no way to *arrive* in-context would leave the best part unused; the decorative sidebar button also already implies this destination, so activating it closes a loop the codebase was already gesturing at. The cost is touching a few existing components (the modal/sidebar and agent surfaces) and their tests, which is contained.

The practical effect: `#contact` becomes the site's shared booking destination, reachable pre-filled from any property or agent context via `?book=`, while `?property=` remains dedicated to the modal. If a future dedicated property route (ADR 0003) ever returns, it can point at the same `?book=` contract without change.
