# Featured Properties' in-page detail view replaces the deferred property detail route for v1

The v1 rescope deferred `/properties/[slug]` as a dedicated page, treating the full property browsing/detail experience as out of scope until its own pass. While designing the Featured Properties section, we walked through what a property card should do on click and ended up reconstructing nearly all of that deferred scope anyway: a full detail view with an image gallery, stats, description, and agent sidebar, addressable by a shareable URL.

Rather than build that content twice - once now as a one-off modal, once later as the "real" route - we're treating the in-page detail view as v1's actual property detail experience, not a stopgap. It opens as a centered frosted-glass overlay panel (`FloatCard` language, consistent with the Reel Stage's established glass identity per ADR 0001) over the grid, instead of navigating to a new page. The open property is reflected in a URL query param (e.g. `?property=[slug]`) so it is shareable and survives a refresh, and the browser back button closes it.

Its content pieces - `PropertyGallery`, `PropertyStats`, `AgentSidebar` - are built as reusable components per the original spec's naming and shape, not modal-only markup. That keeps the door open for an eventual dedicated route to wrap the same components in a page shell, without treating this work as throwaway.

We considered keeping the modal to a lighter "quick-look" summary (image, name, price, stats) to avoid this overlap and leave the detail page genuinely deferred. We chose the fuller version because a Featured Properties section that can't show a property's full detail felt like an unfinished feature, and splitting "preview in a modal" from "real detail on a route" would mean designing and building the same information twice. The practical effect: `/properties/[slug]` is no longer "deferred scope to build later" - it is superseded by this pattern unless a future need (e.g. SEO-indexable property pages, deep links from outside the site) brings it back.
